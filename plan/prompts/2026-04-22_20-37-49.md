# Prompt: Full-Stack Global Leaderboard

**Task:** Transform the local slot machine into a full-stack application by implementing a lightweight backend server and a "Global Leaderboard" in the frontend.

**Objective:**
Create a Node.js/Express backend to store player scores in memory. Update the frontend UI to fetch and display the top 10 scores, and allow users to "Cash Out" by submitting their name and current balance to the global leaderboard.

## Requirements

### 1. Backend Implementation (The Server)
- Create a new directory named `server` at the root of the project.
- Inside `server`, generate a basic `server.js` file using `express` and `cors`.
- **In-Memory Store:** Use a simple JavaScript array to store leaderboard entries (e.g., `[{ name: "AAA", score: 5000, date: "..." }]`).
- **Endpoints:**
  - `GET /api/leaderboard`: Returns the top 10 entries sorted by score in descending order.
  - `POST /api/leaderboard`: Accepts a JSON payload `{ name, score }`, adds it to the array, sorts the array, and keeps only the top 100 records to prevent memory bloat.
- **Note:** Do not use a database like SQLite or MongoDB. Keep it entirely in-memory for this prototype. Run the server on port 3000.

### 2. Frontend UI (The Leaderboard Tab)
- **Drawer Update:** In `src/index.html`, add a new "🏆 LEADERBOARD" button to the Splash Screen and a corresponding tab in the `#side-drawer`.
- **Submit Score UI:** In the drawer (or main UI), add a "Cash Out & Submit Score" section containing:
  - A text input for the player's initials (max 3 characters).
  - A "Submit" button.
- **Scoreboard UI:** Add a container in the Leaderboard tab to render an ordered list of the top 10 scores fetched from the server.

### 3. Frontend Logic (API Integration)
- Create `src/api/apiClient.js` (or integrate directly into `view.js`/`GameManager.js`).
- **Fetch Scores:** Write an async function to `GET http://localhost:3000/api/leaderboard` and populate the Leaderboard UI when the tab is opened.
- **Submit Score:** Write an async function to `POST` the user's initials and current `Wallet` balance. 
  - After a successful submission, reset the user's `Wallet` balance to the default starting amount (e.g., 1000) and refresh the leaderboard UI.

### 4. Constraints
- **Do not break existing features:** The Auto-Spin, Procedural Audio, and Daily Mechanics must remain intact.
- **Error Handling:** If the server is offline, the frontend should catch the `fetch` error and display gracefully (e.g., "Leaderboard unavailable. Playing offline mode.").
- **Styling:** Match the existing "Vegas Gold & Purple" theme. 

## Deliverables
1. `server/package.json` (with express and cors dependencies).
2. `server/server.js` (the backend API).
3. Updated `src/index.html`, `src/style.css`, and `src/ui/view.js` for the new UI.
4. Updated `src/controller/GameManager.js` to handle the "Cash Out" logic.

## Success Condition
Running `node server.js` starts the backend. The frontend can successfully POST a new high score to the server, zero out the local balance, and GET the updated global top 10 list to display in the UI.
