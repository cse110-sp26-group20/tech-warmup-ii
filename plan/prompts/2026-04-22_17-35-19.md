# Increment 19 — Full Linting and Formatting Sweep

## Role

You are a senior software engineer performing a final code quality pass on a modular JavaScript slot machine project.

---

## Context

- The project follows a strict 3-layer architecture:
  1. **Controller (GameManager)** — no DOM access
  2. **Logic modules (Wallet, RNG, math)** — pure/testable
  3. **UI/View** — DOM only, no game logic

- The codebase is **feature-complete and functionally correct**
- Previous increments have implemented gameplay, UI, testing, and auditing
- Your task is strictly **code quality enforcement**, not feature development

---

## Task

Execute a **full linting and formatting sweep** across the entire codebase:

### Scope

- All JavaScript files (`src/`, `tests/`)
- All CSS files
- All HTML files

### Tools (conceptual)

- ESLint (JavaScript)
- Prettier (formatting)
- stylelint (CSS)
- HTMLHint (HTML)

---

## Requirements

### 1. Identify Issues

Scan all files and list:

- Errors (**must fix**)
- Warnings (**fix if safe**)
- Formatting inconsistencies

Group findings by:

- JavaScript
- CSS
- HTML

---

### 2. Apply Fixes

- Automatically fix all safe issues (formatting, minor lint rules)
- Manually fix remaining issues where necessary

---

### 3. STRICT Constraints

- DO NOT change program behavior
- DO NOT refactor architecture
- DO NOT rename files or move modules
- DO NOT modify business logic unless required to resolve a lint error
- DO NOT introduce new dependencies

If a lint rule conflicts with working logic:

- Prefer minimal, non-invasive fixes
- OR justify leaving it unchanged

---

### 4. Code Standards to Enforce

- Consistent indentation and spacing
- Consistent quote style
- Proper semicolon usage (or consistent omission)
- No unused variables or imports
- No unreachable code
- Clear and readable formatting

---

## Output Format

### A. Summary Report

- Total issues found (by type)
- Files affected
- High-level description of fixes

---

### B. Key Fix Examples

Provide **before → after** snippets for representative fixes

---

### C. Manual Decisions

Explain any cases where:

- A rule was ignored
- A fix required judgment

---

### D. Final Status

Confirm:

- Zero lint errors
- Zero formatting inconsistencies
- No behavioral changes introduced

Write the output in the format to a file plan/final-lint-sweep.md

---

## Verification

Explain how to verify results using:

```bash
npm run lint
npm run lint:fix
```
