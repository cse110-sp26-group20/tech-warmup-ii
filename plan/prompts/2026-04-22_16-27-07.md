# Increment 17.5 Prompt

**Task:** Implement ethical fixes and remove dark patterns from the slot machine project.

**Objective:** Address the critical compliance violations identified in the recent ethical audit. This increment focuses on aligning the UI paytable with the true mathematical payouts, removing manipulative retention mechanics (like bankruptcy incentives and escalating streaks), implementing modern safeguards for the auto-spin feature, and increasing overall transparency for the player.

**Requirements:**

1. **Paytable and Odds Transparency (UI):**
   - Update the Paytable tab in `src/index.html` so that the displayed multipliers perfectly match the internal `PAYTABLE` values used in `src/logic/SlotMachineMath.js`.
   - Add a transparent disclosure section in the Paytable tab that explicitly states the theoretical Return to Player (RTP) percentage and the odds of hitting the highest payout symbol.
   - Update the UI to clearly state how the bet math works (e.g., explicitly showing that a bet of $1 applies to 5 lines for a $5 total wager, or splitting the $1 wager to $0.20 per line).

2. **Reward Structure Adjustments (GameManager):**
   - Refactor `checkDailyReward` in `src/controller/GameManager.js` to remove the "Bankruptcy Bonus" (the logic that rewards users specifically for draining their balance to zero).

3. **Auto-Spin Safeguards (GameManager & View):**
   - Implement basic safety parameters for the Auto-Spin feature.
   - Add non-intrusive UI inputs in `src/index.html` and `src/ui/view.js` allowing the user to set a "Stop Loss" limit (stop if balance decreases by X).
   - Add an input for a "Single Win Limit" (stop if a single win exceeds Y).
   - Update `GameManager.js` to evaluate these new limits during the auto-spin loop and halt the sequence if either condition is met.

4. **Testing and Coverage:**
   - Update `tests/GameManager.test.js` to align with the new flat daily bonus logic, removing obsolete tests for streaks and bankruptcy bonuses.
   - Write new unit tests to verify that Auto-Spin halts correctly when the stop-loss limit or single-win limit is reached.
   - Run the full lint and test suite to ensure no regressions.

**Expected deliverables:**

1. Updated `src/index.html` reflecting the true paytable, RTP disclosures, and auto-spin safety inputs.
2. Updated `src/controller/GameManager.js` with the revised flat daily reward logic and auto-spin guardrails.
3. Updated `src/ui/view.js` handling the new inputs and removing streak UI elements.
4. Updated `tests/GameManager.test.js` validating the ethical logic changes.
5. Confirmation that lint passes.
6. Confirmation that the full test suite passes.
7. A short summary stating:
   - Which ethical fixes were implemented.
   - How the auto-spin safeguards were tested.
   - Whether any production code was adjusted.
   - Which files were modified.
   - Whether lint passes.
   - Whether all tests pass.

**Constraints:**

- Keep changes focused strictly on resolving the audit findings; do not alter the underlying core RNG or the internal payout math, only how they are utilized and displayed.
- Ensure the removal of the streak mechanic gracefully handles any existing `localStorage` data without crashing.
- Do not introduce new complex mechanics; replace manipulative systems with flat, simple alternatives.

**Success condition:** The slot machine is ethically compliant. The UI honestly reflects the game's math, auto-spin includes mandatory user safety limits, manipulative bankruptcy/streak rewards are replaced with a flat bonus, and the test suite passes with full coverage of the new guardrails.
