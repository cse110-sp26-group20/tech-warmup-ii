# Increment 12 Prompt: High-Fidelity Procedural Audio

Implement a professional `AudioManager` that uses the **Web Audio API** to procedurally synthesize high-quality slot machine sound effects. The goal is to create a "Vegas" atmosphere without using external MP3 files.

## Requirements

1. **Synthesis Engine:**
   - Create `src/audio/AudioManager.js`.
   - Implement a centralized `AudioContext` and a master gain node for volume control.
   - Use advanced synthesis techniques (Envelopes, Filters, and Oscillators) to create the following SFX:
     - **Spin Sound:** A combination of low-frequency oscillators and filtered white noise to simulate mechanical reel whirring.
     - **Reel Stop:** A percussive "thud" created using a pitch-shifted sine wave with a rapid decay envelope.
     - **Win (Small/Medium):** An FM Synthesis (Frequency Modulation) routine to create bright, metallic bell-like chimes.
     - **Jackpot:** A complex, escalating sequence of chords using additive synthesis (multiple oscillators) and a resonant filter sweep.

2. **Technical Implementation:**
   - **Audio Unlock:** Create a method to resume the `AudioContext` on the first user interaction (click/touch) to bypass browser autoplay restrictions.
   - **Polyphony:** Ensure the "Reel Stop" sound can be triggered rapidly and overlapping without cutting off previous instances (use a new source node for every trigger).
   - **Cleanup:** Ensure nodes are disconnected or stopped after playback to prevent memory leaks.

3. **Architecture Integration:**
   - Connect the `AudioManager` to `src/controller/GameManager.js`.
   - Trigger `spin()` at the start of a spin.
   - Trigger `stopReel()` as each individual reel locks into place.
   - Trigger `playWin(multiplier)` with variable intensity based on the win size.

4. **Deliverables:**
   - `src/audio/AudioManager.js`: The procedural synthesis class.
   - Updated `src/controller/GameManager.js`: Integrated audio triggers.
   - A `toggleMute()` function exported for future UI integration.

## Constraints:
- **No External Files:** Do not attempt to load .mp3 or .wav files; all audio must be generated via code.
- **Ethics:** Ensure win sounds are celebratory and loss sounds are neutral/short. 
- **Performance:** Keep the AudioContext efficient; do not create unnecessary nodes.

## Success Condition:
The game provides rich, multi-layered audio feedback for every action. The sound of the reels stopping is perfectly synchronized with the visual "stop" frame in the UI.
