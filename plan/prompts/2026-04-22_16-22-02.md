Increment 17:

You are a senior product ethics auditor and responsible game design reviewer.

Context:
This is a browser-based slot machine game with simulated currency (no real money).
The system already includes:

- Wallet system
- Auto-spin feature (with stop conditions)
- RTP-based math model (~96%)
- Daily login rewards and streaks
- Audio + UI feedback systems

Task: Perform a full Ethical & Dark Pattern Audit of the entire system.

You must analyze:

1. UI behavior (menus, buttons, animations, feedback loops)
2. Game logic (GameManager, Wallet, Auto-Spin, RTP system)
3. Reward systems (daily login, streaks, bonuses)
4. Audio/visual feedback systems
5. Any persistence mechanisms (localStorage, streak tracking)

---

### Evaluate against these violation categories:

#### 1. Loss of user control

- Can users stop auto-spin immediately?
- Are there forced delays or hidden lockouts?
- Is any system running without explicit consent?

#### 2. Misleading feedback loops

- Do “wins” feel inflated compared to actual value?
- Are near-misses artificially emphasized?
- Does UI exaggerate success states?

#### 3. Manipulative reward structures

- Do streaks pressure daily return?
- Does “bankruptcy bonus” encourage compulsive reset behavior?
- Are rewards escalating in a psychologically coercive way?

#### 4. Obscured probability / RTP confusion

- Is RTP transparent or hidden from users?
- Are outcomes visually misleading relative to actual probability?

#### 5. Forced engagement mechanics

- Any auto-play systems that encourage extended play without friction?
- Any UI patterns that reduce natural stopping points?

---

### Output format:

1. **Executive Summary**
   - Is the system ethically compliant? (Yes / No / Partially)

2. **Violations Found (if any)**
   - Category
   - File/module where it occurs
   - Why it is a problem
   - Severity (Low / Medium / High)

3. **Design Risks**
   - Even if not a violation, flag risky mechanics

4. **Required Fixes**
   - Concrete code or UI changes needed

5. **Safe Design Recommendations**
   - How to improve responsible gameplay without breaking UX

Write the output to a file: plan/audit-ethic-findings.md

---

### Constraints:

- Do NOT change gameplay logic
- Do NOT refactor code
- Only analyze and report
- Be strict and assume worst-case interpretation of ambiguity
