# Increment 7 Prompt

Task: Implement Increment 7 for this slot machine project.

Objective:
Add win feedback visuals and minimal loss feedback using controller-provided result data, ensuring that feedback is accurate, correctly timed after animation, and does not duplicate or recompute game logic.

Requirements:

1. Implement win feedback:
   - clearly highlight winning paylines or positions
   - visually emphasize payout and success state
   - ensure feedback is consistent with controller-provided result data

2. Implement loss feedback:
   - provide minimal but clear indication of a non-winning spin
   - avoid visual clutter or misleading feedback

3. Preserve controller contract:
   - all win/loss determination must come from `GameManager`
   - view must not recompute or infer win conditions

4. Ensure correct timing:
   - feedback must occur after animation completes
   - feedback must not trigger early during spin playback

5. Validate correctness of feedback:
   - highlighted positions must match actual winningLines
   - no false positives or incorrect highlights
   - payout display must match controller output

6. Preserve spin flow sequencing:
   - animation → result reveal → feedback → reset
   - no interference with wallet updates or spin state

7. Handle edge cases:
   - no feedback duplication across spins
   - feedback must clear/reset properly before next spin
   - rapid user interaction must not cause inconsistent UI state

8. Maintain separation of concerns:
   - controller triggers feedback via interface
   - view handles rendering only

9. Keep changes minimal and scoped to feedback only.

10. Do not break existing tests or linting.

Expected deliverables:

1. Enhanced win/loss feedback visuals
2. Correct integration with animation timing
3. Existing app still runs correctly end-to-end
4. A short summary stating:
   - what feedback was added
   - how correctness is ensured
   - how timing is handled
   - which files were modified

Constraints:

- Do not change payout logic
- Do not change slot math
- Do not compute wins in the view
- Do not redesign UI
- Do not add new gameplay features
- Do not introduce unrelated refactors

Success condition:
Winning spins correctly highlight outcomes and payouts, losing spins provide minimal feedback, and all feedback is accurate, correctly timed, and driven by controller data.
