# Increment 20

You are a senior JavaScript engineer working on a slot machine project.

Context:
- The slot machine game is already fully implemented and production-ready.
- It uses an existing RNG/maths system to determine spin outcomes.
- We need a temporary development-only script to force deterministic outcomes for recording a presentation video.

Task:
Create a standalone script that can be executed during development to override RNG behavior and force a predefined sequence of spin outcomes.

---

## Objective

Build a lightweight script that:
- Forces specific spin outcomes in a fixed order
- Temporarily overrides or monkey-patches the existing RNG / math outcome function
- Requires NO permanent changes to game architecture
- Can be turned on/off easily for testing and recording

---

## Requirements

### 1. Script location
Create a standalone file such as:
- scripts/force-demo-outcomes.js

This script should be runnable manually (e.g. via node or injected in dev tools).

---

### 2. Behavior override
The script must:
- Hook into the existing RNG or spin outcome function
- Replace it temporarily with a deterministic sequence generator

Example sequence:
- Spin 1 → small win
- Spin 2 → loss
- Spin 3 → medium win
- Spin 4 → near miss
- Spin 5 → jackpot

After sequence ends:
- Either repeat or fallback to original RNG (configurable)

---

### Non-invasive design
- Do NOT modify core game files permanently
- Do NOT introduce new architecture or dependencies
- Use safe monkey-patching or function wrapping only
- Must be reversible (restore original function after use)

---

### Developer usability
Include:
- A simple enable/disable toggle
- A clear console log indicating demo override is active
- A way to reset sequence index

---

## Constraints

- Must NOT change Wallet, GameManager, or payout logic
- Must NOT introduce production code dependencies
- Must NOT affect normal gameplay when script is not executed
- Must remain strictly a dev tool

---

## Output Requirements

1. Full script implementation (single file)
2. Explanation of what function(s) it overrides
3. How to run it (node or browser injection method)
4. How to disable and restore original behavior
5. Example output sequence behavior in console logs

---

## Purpose

This script exists ONLY to guarantee deterministic gameplay sequences during recording of the final presentation video.
