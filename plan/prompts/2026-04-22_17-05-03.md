# Increment 18

You are a senior JavaScript systems engineer working on a slot machine simulation. The slot machine game already has a fully implemented math/RNG system. Game logic and payout calculations are stable and tested.

## Objective
- The target Return To Player (RTP) is 96%.
- We need to statistically validate this using large-scale simulation.

---

## Task

Implement a simulation script at:

    tests/rtp-sim.js

This script must run a high-volume Monte Carlo simulation of the slot machine.

---

##  Requirements

### 1. Simulation scale
- Run at least 500,000 spins
- Each spin must use the EXISTING math model (do NOT reimplement logic)

---

### 2. Metrics tracking
The script must accurately track:

- total wagered amount
- total returned amount
- calculated RTP percentage:
  RTP = (total_returned / total_wagered) * 100

---

### 3. Performance + logging
- Log progress every 50,000 spins
- Ensure logging does NOT significantly slow down execution
- Must remain memory-safe over full run

---

## Constraints

- DO NOT modify GameManager, Wallet, or UI code
- DO NOT reimplement spin logic
- DO NOT duplicate payout formulas
- DO NOT introduce browser/UI dependencies
- Script must be fully headless (Node.js or equivalent runtime)

---

## Output Requirements

1. Full implementation of `tests/rtp-sim.js`
2. Explanation of how it interfaces with the existing math/RNG system
3. Example console output during execution (progress + final RTP)
4. Final interpretation:
   - Is RTP close to 96%?
   - Define acceptable variance range (statistical tolerance for 500k spins)
5. Edge case handling:
   - zero wager protection
   - invalid or undefined spin results
   - numerical stability over large sums

---

##  Success Criteria

The simulation is considered successful if:
- RTP converges close to 96% as spin count increases
- No crashes or memory leaks during 500,000 spins
