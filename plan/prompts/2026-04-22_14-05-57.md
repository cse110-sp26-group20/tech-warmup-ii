# Increment 11 Prompt

Implement high-fidelity adjustable bet controls in the slot machine UI. This increment focuses on a "High Roller" experience by allowing bets up to the total available balance and improving UX through a "Hold-to-Repeat" interaction model.

## Requirements

1. **UI Elements:**
   - Add/Verify interactive buttons for increasing and decreasing the bet.
   - Add a display element to show the current bet amount.
   - Ensure these elements reside in the bottom-heavy controls area as per the `ui-ux` skill, maintaining the "Vegas Gold & Purple" theme.

2. **UI Logic (Rapid Adjustments):**
   - Implement **"Hold-to-Repeat"** functionality for both buttons:
     - On `mousedown` or `touchstart`, trigger a single adjustment.
     - If held for more than 500ms (long-press), begin repeating the adjustment every 100ms.
     - **CRITICAL:** Ensure all timers/intervals are cleared on `mouseup`, `mouseleave`, or `touchend` to prevent "runaway" bet increases.
   - Update the UI display in real-time as the user holds or clicks the controls.

3. **Game Manager Logic (Dynamic Scaling):**
   - **Dynamic Limits:** Remove any hard-coded maximum bet (e.g., 10). The maximum allowed bet must be dynamically tied to the current `Wallet` balance.
   - **Validation:** - Minimum bet remains 1.
     - If a user attempts to increase the bet beyond their total balance, stop the adjustment and display a "Not enough balance!" message in the status area.
   - Ensure the `isSpinning` guard prevents any adjustments while a spin is in progress.

4. **Integration and Validation:**
   - Ensure the `GameManager` correctly passes the updated `currentBet` to the `Wallet` and `SlotMachineMath` modules during the spin execution.
   - Feedback: Provide subtle visual or text feedback when the user hits the minimum (1) or their personal maximum (total balance).

## Deliverables

- Updated `src/ui/view.js` containing the timer/interval logic for the hold-to-repeat feature.
- Updated `src/controller/GameManager.js` with dynamic bet validation linked to the Wallet balance.
- Verified functionality where a user can rapidly scale their bet from 1 to their total bankroll by holding the '+' button.

## Constraints

- **Logic:** Do not change the core RNG or payout evaluation in `SlotMachineMath.js`.
- **State:** Do not change how the `Wallet.js` persists data to `localStorage`.
- **Architecture:** Maintain the strict separation of concerns; the `View` handles the timers, while the `Manager` handles the validation and state.
- **Refactors:** Do not introduce unrelated changes to the spin animation or sound systems.

## Success Condition

Users can intuitively and rapidly adjust their bet using either single taps or long-press interactions. The system accurately prevents bets from exceeding the user's real-time balance without using an artificial maximum cap.
