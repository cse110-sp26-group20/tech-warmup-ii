# Increment 8 Prompt

Task: Implement Increment 8 for this slot machine project.

Objective:
Apply cohesive thematic styling across the UI, improving visual consistency and hierarchy while preserving layout, behavior, and compatibility with animation and feedback.

Requirements:

1. Refine styling across all UI zones:
   - top bar (balance, status)
   - center reels (grid)
   - bottom controls (bet/spin)

2. Improve visual hierarchy:
   - ensure important information (balance, bet, status) is clearly readable
   - ensure controls are visually distinct and usable

3. Improve layout consistency:
   - align spacing, margins, and positioning
   - eliminate inconsistent sizing or misalignment

4. Enhance visual cohesion:
   - consistent color usage
   - consistent borders, backgrounds, and typography
   - unified theme across all components

5. Preserve structure:
   - do not significantly change HTML layout
   - maintain 3-zone architecture

6. Ensure compatibility:
   - animation must still render correctly
   - win/loss feedback must still display correctly

7. Avoid introducing UI regressions:
   - no overlapping elements
   - no broken layouts
   - no reduced readability

8. Keep styling maintainable:
   - avoid overly complex CSS
   - avoid duplication

9. Keep changes scoped to styling only.

10. Do not break existing tests or linting.

Expected deliverables:

1. Updated CSS and any minimal supporting HTML adjustments
2. Improved visual consistency across the app
3. Existing app still runs correctly end-to-end
4. A short summary stating:
   - what styling improvements were made
   - what issues were corrected
   - which files were modified

Constraints:

- Do not change gameplay logic
- Do not change controller behavior
- Do not redesign entire UI
- Do not add new features
- Do not introduce unrelated refactors

Success condition:
The UI is visually consistent, readable, and cohesive, with no regressions in layout, animation, or feedback behavior.
